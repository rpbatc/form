How it works
The Navy formula gives a calibrated BF% estimate based on circumference measurements — more reliable than bioelectrical impedance scales, which vary with hydration.

Your scale correction is calculated by comparing the Navy result to your scale reading on the same day. This offset is then applied to all your daily scale readings to estimate actual BF%.

Setup

Google Drive sync · GitHub Pages hosting

Part 1 — Google Drive sync
Your data is stored in a private file in your Google Drive's app data folder (invisible in Drive UI, private to this app only).

Step 1 — Go to console.cloud.google.com → New Project → name it "Form Physique"
Step 2 — APIs & Services → Enable APIs → search "Google Drive API" → Enable
Step 3 — Credentials → Create Credentials → OAuth Client ID → Web application
  · Authorised JavaScript origins: add your GitHub Pages URL e.g. https://yourusername.github.io
  · Also add http://localhost for local testing
  · Copy the Client ID (ends in .apps.googleusercontent.com)
Step 4 — OAuth consent screen → External → add your email as test user → add scope drive.appdata
Step 5 — Open index.html in a text editor, find const GOOGLE_CLIENT_ID = 'YOUR_GOOGLE_CLIENT_ID' and paste your Client ID
Step 6 — Open the app from your GitHub Pages URL → click Connect Drive → authorise → done ✓
Part 2 — GitHub Pages hosting
Step 1 — Go to github.com → Sign up or sign in
Step 2 — Click + → New repository → name it form → set Private → ✓ Add README → Create
Step 3 — Add file → Upload files → upload this file renamed as index.html → Commit
Step 4 — Settings → Pages → Source: Deploy from branch → main / root → Save
Step 5 — Wait ~60s → your URL: https://yourusername.github.io/form
On iPhone — open URL in Safari → Share → Add to Home Screen → app icon with no browser chrome
To update — GitHub repo → index.html → Edit pencil → paste new code → Commit. Live in ~30 seconds.
Privacy
· GitHub Pages serves static files only — no server ever sees your data
· Drive data sits in your own appDataFolder — only this app can read it
· Keep your repo Private so only you can see the source
· Your OAuth Client ID is safe to embed — scoped to appdata only
· localStorage always acts as a local backup regardless of Drive status
